3.2 Thuật toán Quay số
#
Nhiệm vụ
Chi tiết
Mức độ
3.2.1
DrawService
Thuật toán quay với SecureRandom + cumulative probability
3.2.2
Trừ lượt — Atomic UPDATE
UPDATE USER_TURN SET remainTurns = remainTurns - 1 WHERE remainTurns > 0 — chống race condition
⭐
3.2.3
Lọc giải theo maxWinsPerUser
Đếm số lần user đã trúng từng giải, tự động bỏ qua giải đã đạt giới hạn
3.2.4
Pessimistic Lock 1 dòng
findByIdWithPessimisticLock(prizeId) — chỉ lock dòng Prize đã random trúng
⭐
3.2.5
Lock timeout 3 giây
@QueryHints(lock.timeout=3000) — quá hạn → fallback trượt, không treo hệ thống
🔴
3.2.6
Double-check sau lock
Sau lock: kiểm tra lại globalRemainQuantity > 0 → nếu hết → fallback giải Trượt
🔴
3.2.7
Trừ kho toàn cục
UPDATE PRIZE SET globalRemainQuantity = globalRemainQuantity - 1 trong cùng transaction
3.2.8
Null-safe fallback giải Trượt
getOrCreateLosePrize() — nếu DB trả null → tạo giải ảo trong RAM, không để NullPointerException
🔴
